# AI 工作字典 V3 编辑词条设计文档

## 目标

V3 要在 V2 的基础上增加“编辑已有词条”的能力。

V2 已经支持：

- 新增词条
- 删除词条
- 标记学习状态：`不会`、`在学`、`已会`
- 搜索、分类筛选、学习状态筛选
- 保存到 `localStorage`
- 重置默认词库

V3 新增：

- 每张词条卡片增加 `编辑` 按钮
- 点击编辑后，把词条内容带入现有表单
- 表单进入编辑模式
- 用户可以修改词条名称、分类、解释、解决的问题
- 保存后更新该词条，并写入 `localStorage`
- 用户可以取消编辑，取消后不改变原词条
- README 补充 V3 说明

## 用户体验

每张词条卡片上会有两个操作按钮：

- `编辑`
- `删除`

用户点击 `编辑` 后：

1. 页面上方的词条表单会填入当前词条内容
2. 表单标题从 `新增学习词条` 变成类似 `编辑词条`
3. 提交按钮文字从 `新增词条` 变成 `保存修改`
4. 显示 `取消编辑` 按钮
5. 用户修改内容后点击 `保存修改`
6. 页面更新该词条并保存到 `localStorage`

用户点击 `取消编辑` 后：

1. 表单清空
2. 表单回到新增模式
3. 原词条不发生变化

## 状态设计

`app.js` 里会增加一个编辑状态：

```text
editingTermId
```

含义：

- `null`：当前是新增模式
- 某个词条 `id`：当前正在编辑这个词条

表单提交时：

- 如果 `editingTermId` 是 `null`，执行新增词条
- 如果 `editingTermId` 有值，执行保存编辑

## 编辑保存逻辑

保存编辑时：

1. 校验表单必填项
2. 找到 `id` 等于 `editingTermId` 的词条
3. 更新：
   - `term`
   - `category`
   - `categoryLabel`
   - `definition`
   - `solves`
4. 保留：
   - `id`
   - `status`
   - `isDefault`
5. 保存完整词条列表到 `localStorage`
6. 表单回到新增模式
7. 重新渲染页面

保留 `status` 的原因：

```text
用户编辑词条内容，不应该丢失“不会 / 在学 / 已会”的学习状态。
```

保留 `id` 的原因：

```text
id 是这张卡片的身份标识。编辑内容不等于换一张新卡片。
```

## 取消编辑逻辑

取消编辑时：

1. 清空表单
2. 把 `editingTermId` 设为 `null`
3. 表单标题和按钮文字回到新增模式
4. 不保存任何数据
5. 不改变原词条

## 表单校验

编辑模式和新增模式使用同一套校验：

- 词条名称不能为空
- 解释不能为空
- 解决的问题不能为空
- 分类必须有效

如果校验失败：

- 显示中文提示
- 不新增
- 不保存编辑

## 风险和处理方式

风险：

- 用户不知道自己正在编辑哪张卡片
- 保存编辑时误把旧词条变成新词条
- 取消编辑后表单状态没有恢复
- 编辑默认词条后重置行为让用户困惑

处理方式：

- 表单区域显示编辑模式提示
- 使用 `editingTermId` 区分新增和编辑
- 保存编辑时保留原词条 `id` 和 `status`
- 取消编辑明确恢复新增模式
- README 说明：点击重置会恢复默认词库，并清空所有自定义修改
