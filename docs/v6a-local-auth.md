# V6A：本地模拟登录

V6A 的学习主题是：账号登录的基本概念。

这一版不会做真实商业登录。它只在浏览器本地保存一个“当前用户”，用来学习：

- 用户是谁；
- 登录状态是什么；
- session 为什么存在；
- 不同用户的数据为什么要隔离；
- 本地模拟登录和真实后端登录有什么区别。

## 设计选择

V6A 使用本地模拟账号：

- 用户输入一个显示名称即可进入学习空间；
- 登录状态保存到 `localStorage`；
- 不同用户名使用不同的词条存储 key；
- 退出登录只清除当前 session，不删除词条数据；
- 没有密码、没有服务器、没有数据库。

## 为什么不直接做真实登录

真实登录需要后端和数据库，还要处理密码哈希、token、安全策略、隐私和部署。对学习路线来说，直接做真实登录会一次引入太多概念。

所以 V6A 先学习“登录是什么”，V6B 或 V7 再进入后端 API 和数据库。

## 真实产品限制

本地模拟登录不能用于真实客户账号安全：

- 用户名不是身份验证；
- localStorage 可以被本机用户查看或清除；
- 没有密码保护；
- 换电脑不会自动同步；
- 不能防止别人冒用同一个名字。

它适合教学，不适合作为生产级账号系统。
