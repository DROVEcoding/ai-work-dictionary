# V9A：登录入口和身份边界

V9A 的学习主题是：身份入口。

V6A 学过本地模拟登录，V7 学过真实云端账号。到了 V9 权限系统之前，需要先把“登录是谁”和“进入应用后做什么”分开。

## 这一版做了什么

- 首页先显示登录入口。
- 用户可以选择云端登录、注册云端账号，或访客模式。
- 删除首页里的旧本地学习账号区域。
- 主应用顶部显示当前身份。
- 云端同步区域只负责上传、下载和退出云端，不再承担登录表单。

## 为什么要这样改

权限系统依赖清楚的身份边界。

如果登录、同步、管理词条都混在同一个页面区域，后面做管理员、普通用户、公共词库时会越来越乱。

V9A 先把入口整理清楚：

```text
登录入口 -> 确认身份 -> 进入主应用 -> 按身份控制权限
```

## 还没做什么

- 还没有管理员角色。
- 还没有公共词库。
- 还没有 Supabase profile 表。
- 还没有真正的角色权限策略。

这些属于 V9B。
