# V12C：超级管理员组织目录

V12C 的学习主题是：后台数据表格、搜索索引和组织详情页。

V12B 已经建立了超级管理员后台，但它更像“能看到全局信息的第一版”。V12C 要把后台往真实 SaaS 方向推进：让超级管理员可以更快地查找用户、组织和反馈，并点开某个组织查看详细关系。

这里继续坚持一个边界：V12C 是只读后台。它允许超级管理员看清楚全局数据，但不允许删除组织、移除成员、转移 owner 或封禁用户。

## 这一版做什么

- 把后台列表升级成表格形式。
- 给后台增加搜索框和筛选器。
- 给数据库增加常用查询索引。
- 增加组织详情区域。
- 超级管理员可以从组织表选择一个组织，查看该组织的成员、拥有者、相关反馈和词库备份摘要。

## 为什么要做表格

后台管理和普通产品页面不一样。

普通产品页面更关心“完成一个任务”；后台更关心“快速扫描、比较、定位异常”。所以真实后台常用表格：

- 一行代表一个对象，比如一个用户或一个组织。
- 一列代表一个字段，比如邮箱、角色、成员数量、创建时间。
- 表格更适合搜索、筛选、排序和批量查看。

V12C 的表格是学习版，不做复杂虚拟滚动和分页。真实数据量很大时，分页和服务端搜索会成为后续版本的重点。

## 搜索和索引的区别

这一版会同时学习两个概念：

- 前端搜索：页面拿到数据后，在浏览器里按关键词过滤，比如搜索邮箱或组织名。
- 数据库索引：数据库提前给常用字段建立“目录”，比如 `profiles.email`、`organizations.name`、`feedback_reports.status`。

可以把数据库索引理解为书后面的索引页。没有索引时，数据库可能要从头翻到尾；有索引后，它能更快定位。

## 组织详情

组织表里每一行会有一个“查看详情”动作。

选中组织后，后台显示：

- 组织名称。
- 创建时间。
- owner 邮箱列表。
- member 邮箱列表。
- 所有成员的邮箱、角色、加入时间。
- 该组织相关反馈。
- 组织词库备份摘要，比如是否存在备份、最后更新时间、词条数量。

## 暂时不做什么

V12C 不做这些高风险操作：

- 删除组织。
- 移除组织成员。
- 转移组织 owner。
- 修改用户平台角色。
- 封禁用户。
- 批量操作。

这些功能不是“按钮加上去”就安全。真实产品需要：

- 操作审计。
- 二次确认。
- 恢复和回滚策略。
- 更细的后台权限，比如客服、运营、技术管理员。
- 明确业务规则，比如最后一个 owner 能不能被移除。

## 数据设计

新增 SQL 文件：

```text
docs/v12c-admin-organization-directory-schema.sql
```

它会做两类事情：

1. 增加数据库索引：
   - `profiles.email`
   - `profiles.role`
   - `organizations.name`
   - `feedback_reports.status`
   - `feedback_reports.organization_id`
   - `organization_memberships.organization_id`
   - `organization_memberships.user_id`

2. 增加组织详情只读 RPC：
   - `get_admin_organization_detail(org_id uuid)`

这个 RPC 必须检查 `public.is_admin()`。不是超级管理员时，数据库直接拒绝读取。

## 前端模块设计

继续使用 V12B 的文件：

- `admin.html`：后台页面结构。
- `scripts/admin.js`：后台页面交互和渲染。
- `scripts/adminData.js`：后台数据读取和格式转换。
- `style.css`：后台表格、筛选器和详情区域样式。

V12C 会新增或扩展：

- 后台搜索状态。
- 后台表格渲染函数。
- 组织详情数据读取函数。
- 组织详情渲染函数。

## 页面结构

后台页面分成四个主要区域：

1. 顶部：身份、返回词典、刷新数据。
2. 总览：用户数、组织数、反馈数、未处理反馈数。
3. 数据表格：用户表、组织表、反馈表。
4. 组织详情：选择组织后显示。

表格上方会有搜索输入框：

```text
搜索用户邮箱、组织名、反馈内容
```

反馈表会有状态筛选：

```text
全部 / open / triaged / resolved
```

## 测试计划

自动测试：

- 用户搜索能按邮箱过滤。
- 组织搜索能按组织名或 owner 邮箱过滤。
- 反馈搜索能按内容、提交人或组织名过滤。
- 反馈状态筛选能只显示指定状态。
- 组织详情数据能转换成前端结构。

手动测试：

1. 管理员打开后台页。
2. 用户表、组织表、反馈表显示为表格。
3. 输入关键词后，表格数量减少并匹配关键词。
4. 切换反馈状态筛选，反馈表随之变化。
5. 点击某个组织的“查看详情”，能看到成员、反馈和词库备份摘要。
6. 普通用户仍然不能进入后台。

## 真实 SaaS 还差什么

V12C 仍然是学习版。真实后台还会继续做：

- 服务端分页。
- 服务端搜索。
- 列排序。
- 导出 CSV。
- 后台操作审计。
- 更细的后台角色权限。
- 危险操作的二次确认和恢复方案。
