# V10E：审计日志

V10E 的学习主题是：安全可追踪性。

权限系统不只要能控制“谁能做什么”，还要能回答“谁在什么时候做了什么”。这就是 `audit log` 审计日志。

## 这一版做什么

- 新增 `organization_audit_logs` 表。
- 添加成员时写入 `member_added` 记录。
- 升级成员时写入 `member_promoted` 记录。
- 移除成员时写入 `member_removed` 记录。
- 页面显示当前组织最近 20 条操作记录。
- 组织成员都可以查看本组织审计日志。

## 关键概念

- `actor`：操作者，比如谁点击了添加成员。
- `target`：被操作对象，比如被添加、被升级、被移除的成员。
- `event_type`：事件类型，比如 `member_added`。
- `details`：事件细节，用 JSON 保存额外信息。
- `traceability`：可追踪性，出问题时能回看发生过什么。

## 为什么重要

真实 SaaS 里，权限操作没有日志会很危险：

```text
某个成员突然变成 owner，谁操作的？
某个成员被移除，什么时候发生的？
组织数据出问题，最近谁做过权限变更？
```

审计日志让系统从“能操作”升级成“能解释发生过什么”。

## 你需要做什么

在 Supabase SQL Editor 执行：

```text
docs/v10e-audit-logs-schema.sql
```

执行成功后：

1. 用 owner 账号登录。
2. 添加一个成员。
3. 升级一个成员。
4. 移除一个普通成员。
5. 查看组织面板里的 `最近操作记录`。

## 真实 SaaS 还差什么

V10E 是最小审计日志。后续可以继续学习：

- 日志分页；
- 按事件类型筛选；
- 按操作者筛选；
- 导出审计日志；
- 关键操作写入不可变日志；
- 更详细的对象快照，比如修改前和修改后的角色。
